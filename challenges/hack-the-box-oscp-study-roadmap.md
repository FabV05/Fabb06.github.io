### Documentation Standards

* Each completed box **must have a dedicated writeup**
* Writeups should include:
  * Initial enumeration methodology
  * Attack path justification
  * Privilege escalation vector
  * Alternative exploitation paths (if applicable)
  * Key takeaways
* Use consistent tagging:
  * `linux`, `windows`, `active-directory`, `prolab`
* Avoid copy-paste exploitation; prioritize reasoning and decision-making

***

### Objective

The objective of this roadmap is not box completion speed, but:

* Pattern recognition across environments
* Deep understanding of Windows Active Directory abuse
* Preparation for advanced certifications (OSEP, CRTO, Red Team engagements)
* Development of a personal, reusable exploitation playbook
